A Secure Exam Scheduling & Notification Management System that includes end-to-end workflows for K–12 and district testing offices to plan exam sessions, manage roster data, and keep stakeholders informed in a fully offline environment. The Vue.js web interface provides role-based navigation for Administrator, Academic Affairs Coordinator, Homeroom Teacher, Subject Teacher, and Student, with each user seeing only the menus and records permitted for their grade/class/course/term scope. Staff users can create and maintain candidate rosters, subject sessions, campuses and exam rooms, and proctor assignments with guided forms, inline validation, and immediate, readable error feedback; bulk import/export supports CSV/XLSX with a preview step that highlights duplicates and invalid formats before committing changes. Version history is visible on each record so staff can compare “as of” snapshots for prior terms and restore an earlier version when needed. Users can publish key events such as schedule changes, review outcomes, and check-in reminders; students receive clear on-screen reminders and a personal subscription settings page to opt in/out by event type, set a Do Not Disturb window (for example, 9:00 PM–7:00 AM), and view delivery status. Where a local WeChat integration is deployed within the organization’s intranet, the system can send subscription-message notifications for deadlines and results; if messaging is not available, the same notifications appear in an in-app inbox so operations remain complete without internet access.

The backend uses Spring Boot to implement REST-style APIs consumed by the Vue.js client, enforcing decoupled authentication, authorization, and data validation at the service layer; MySQL persists all master data, schedules, audit logs, and notification/outbox records. Authentication is strictly local username plus password (no SMS, email codes, or third-party login); passwords must be at least 12 characters with complexity rules, stored with salted hashing, and sessions expire after 30 minutes of inactivity with optional “remember device” limited to 7 days on managed machines. RBAC/ABAC checks are applied on every request, including row-level data scope and action permissions such as view/enter/import/review/publish; suspicious-login protection locks an account for 15 minutes after 5 failed attempts and blocks concurrent sessions from different devices unless explicitly allowed by Admin. Security and risk controls include sensitive-content and minor-protection safeguards on training prompts and health-related disclaimers, plus an internal compliance review queue that must approve publishable content before student visibility. All APIs require request signing with timestamp and nonce to prevent replay (reject if older than 120 seconds), plus rate limiting (for example, 60 requests/minute/user and 300/minute/IP) and server-side input sanitization; sensitive fields like student ID are masked in UI and encrypted at rest. Distributed job scheduling runs on multiple nodes with sharded queues for notification sends, bulk imports, and periodic data checks; jobs are idempotent using dedup keys, retry up to 3 times with exponential backoff, and expose a visual monitor for status, failures, and manual re-run. Anti-cheat and leaderboard-fraud detection is applied to any ranking-style views by flagging impossible activity bursts, repeated identical submissions, and abnormal score deltas for human review rather than automated punitive actions.
